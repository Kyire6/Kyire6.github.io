部署项目到Nginx服务器

发表于 2020-06-18 更新于 2020-07-12 分类于 实训日志 阅读次数： 评论数：
本文字数： 705 阅读时长 ≈ 1 分钟

前言

实训第四天，上午

Nginx服务器的项目部署

一、将客户端上传到服务器

1. 安装putty文件上传工具

​ 下载地址：官网下载

​ 下载好安装包，双击打开进行安装，一直next即可

2. 使用putty工具上传和下载文件

​ 上传文件

​ pscp 本地文件路径 root@公网IP:路径

​ 输入密码后上传成功

​ 上传文件夹

​ pscp -r 本地文件夹路径 root@公网IP:路径

​

​ 下载文件

​ pscp root@公网IP:文件路径 本地路径

​ 下载文件夹

​ pscp -r root@公网IP:文件夹路径 本地路径

3. 检查服务器上是否有上传文件

​ 检查要访问的默认文件夹是不是在nginx的配置文件中的默认根目录(/opt/www)

​

​ 如果不在，有两种解决办法

​ 1.修改配置nginx的配置文件中root目录（加一个目录root /opt/www/web）

​

​ 2.将上传的资源移动到nginx.com配置文件中的root目录中

4. 重新加载nginx配置资源

​ service nginx reload

5. 打开浏览器访问

可以看到现在还查询不到疫情数据，因为服务端还没配置好

下一步就是配置服务端，使之能成功查询到疫情数据

​

二、将服务端上传到服务器

1. 上传服务端文件

2. 修改客户端访问数据地址为服务器的IP地址

/opt/www/web/index.html

3. 安装NodeJS

apt install nodejs

输入node检查是否安装成功：

4. 启动服务端

进到服务端的上传文件夹，用nodejs启动服务端

/opt/www/server/index.js

node index

5. 在阿里云实例安全组配置新规则

手动添加8090端口的访问规则

6. 浏览器中访问

可以看到成功查询到了疫情数据
